1. Introduction

This Software Design Document provides information on the group project for CP-317-C at Wilfrid Laurier University. This project will process two text files, where one of the files contains the students names, and the other with the course along with the test scores, and then output to a new file summarizing the students performance. This software contains two input files, “CourseFiles.txt” and “NameFile.txt”. CourseFile includes Student ID, Course codes, Test scores, and Exam scores. NameFile includes the student ID and student name. This software will calculate the student's final grade in their course by using the formula: 

Final Grade = Test 1 x 20% + Test 2 x 20% + Test 3 x 20% + Final Exam x 40%

Then, formatted and output results into a new file, which is sorted by student ID. This project is completed by a group of four students in CP-317-C and serves as a tool to help develop our skills in object-oriented design principles, offensive programming, communication and collaboration skills.

2. Architecture design/System design

Monolithic

This software application follows a monolithic architecture, meaning the core functionalities are bundled into a single program. All operations including file reading, data processing, and output generation occur within one integrated system.

2.1 System Components

  1. Input Module
    ○ Reads and parses data from NameFile.txt and CourseFile.txt.
    ○ Uses file I/O with exception handling to validate data format.
  2. Data Processing Module
    ○ Matches student IDs from both files.
    ○ Calculates final grades using:
    Final = (Test1 + Test2 + Test3) * 0.2 + FinalExam * 0.4
    ○ Stores processed data in an internal list or class object.
  3. Sorting Module
    ○ Sorts the final student data by student ID.
    ○ Uses Java’s Collections.sort() or equivalent algorithm.
  4. Output Module
    ○ Writes results into StudentDatabase.txt or similar.
    ○ Formats output with one decimal place for grades.

3. Detailed design

Technical Specifications

Information about programming languages, frameworks, libraries, and other technologies used in the development process.

3.1 Overview

This software will be using Java as our main compiling language, and the purpose of this program is to calculate and format students final grades into a new output file. It will read the files, “NameFile.txt” and “CourseFile.txt,” and calculate the final grades. Then a new file will be created and sorted by student ID and course code.

3.2 Software Calculation

Final grade calculation will be made by reading the file, “CourseFile.txt,” and splitting each line by “,”. This file will take the last four elements, which are the grades for Tests 1-3 and then the final exam. The first three of those elements will be the tests, which will be calculated by using the formula Test 1 x 20% + Test 2 x 20% + Test 3 x 20%. After that has been calculated, the final exam grade will be added too, Final Exam x 40% + Test Grades.

Once the final grades have been calculated, the student ID will be compared to the other file, “NameFile.txt”. If the Student IDs match, a new array will be created, the first element includes the students ID, the second element includes the student's name, the third element includes the course code, and the fourth element includes the final grade.
